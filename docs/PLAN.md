# Smart Clinic — الخطة المعمارية الشاملة (Roadmap)

> نظام متكامل لإدارة عيادات الأسنان والمراكز الطبية.
> هذا المستند هو المرجع الرئيسي للبناء قبل كتابة أي كود.

---

## 1. التقنية المختارة (Tech Stack) ولماذا

بعد مقارنة الخيارات، التوصية النهائية هي:

| الطبقة | التقنية | السبب |
|--------|---------|-------|
| **Framework** | **Next.js 15 (App Router) + TypeScript** | Full-stack في مشروع واحد (واجهة + API)، أسرع تطوير، دعم ممتاز للسحابة (Vercel/Docker)، تكامل سهل مع كل الخدمات. |
| **قاعدة البيانات** | **PostgreSQL** | قوية، موثوقة، مجانية، تدعم العلاقات المعقدة المطلوبة (مرضى/حجوزات/مدفوعات/مخزون). |
| **ORM** | **Prisma** | Type-safe، هجرات (migrations) منظمة، يقلل الأخطاء بشكل كبير. |
| **الواجهة (UI)** | **Tailwind CSS + shadcn/ui** | تصميم حديث، دعم RTL (عربي) كامل، مكوّنات جاهزة احترافية. |
| **المصادقة** | **NextAuth (Auth.js)** | أدوار متعددة (مالك/طبيب/سكرتارية/مريض)، آمن. |
| **المساعد الذكي** | **Claude API (Anthropic)** | يقرأ ملف المريض ويجاوب، أوامر صوتية، إنشاء روشتات وتقارير. |
| **واتساب** | **WhatsApp Cloud API (Meta)** | رسائل، ملفات (صور/فيديو/PDF)، جدولة، تتبع الحالة. |
| **المهام المجدولة** | **node-cron / BullMQ (Redis)** | جدولة رسائل الواتساب والتذكيرات (Reminders). |
| **الاستضافة (Cloud)** | **Docker + أي مزود سحابي** | وصول من أي جهاز (موبايل/تابلت/لابتوب)، تصميم Responsive. |

### لماذا Next.js وليس Laravel/Django؟
- **تكامل أفضل**: الواجهة والباك-إند بنفس اللغة (TypeScript) = أخطاء أقل وسرعة أكبر.
- **سحابي بالفطرة**: مصمم للنشر السحابي والوصول من أي مكان.
- **AI-friendly**: تكامل سلس مع Claude API للمساعد الذكي والأوامر الصوتية.
- **PWA**: يمكن تحويله لتطبيق موبايل يعمل offline جزئياً.

## 3. الأدوار والصلاحيات (Roles)

| الدور | الصلاحيات |
|------|-----------|
| **Owner / Admin** | كل شيء + إدارة الفروع والأطباء والتقارير المالية. |
| **Doctor (طبيب)** | ملفات المرضى، التشخيص، الروشتات، Dental Chart، الحجوزات الخاصة به. |
| **Receptionist (سكرتارية)** | الحجوزات، المدفوعات، استقبال المرضى، الواتساب. |
| **Patient (مريض)** | الحجز الإلكتروني، عرض مواعيده ومدفوعاته وملفاته. |

## 4. نموذج قاعدة البيانات (Data Model — الكيانات الأساسية)

```
Branch (فرع)
  └── Doctor (طبيب)  ──┐
                      ├── Appointment (حجز) ── Patient (مريض)
User (مستخدم/دور) ────┘

Patient (مريض)
  ├── MedicalHistory (تاريخ طبي)
  ├── Diagnosis (تشخيص)
  ├── Prescription (روشتة) ── PrescriptionItem (دواء + بدائل)
  ├── DentalChart (مخطط الأسنان) ── ToothRecord (سن + إجراء + سعر)
  ├── Appointment (حجوزات)
  ├── Payment (مدفوعات) ── Invoice (فاتورة)
  ├── PatientFile (ملفات: صور/أشعة/PDF)
  ├── Complaint (شكوى)
  ├── Package (باقة) ── PackageSession (جلسات)
  └── MedicalReport (تقرير طبي)

Procedure (إجراء طبي: حشو/خلع/تنظيف) ── سعر
  └── ProcedureProduct (ربط المنتج بالإجراء) ── خصم تلقائي من المخزون

Product (منتج/مخزون) ── StockMovement (حركة مخزون)

WhatsAppMessage (رسالة) ── status / scheduledAt / repeat / attachments
Reminder (تذكير) ── الباقات والمواعيد

PrescriptionTemplate (نموذج روشتة محفوظ)
```
